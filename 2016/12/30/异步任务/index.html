#Android异步任务
在我们的程序当中，难免会用的异步任务，因为如果我们的程序在主线程当中运行了耗时的操作，就会阻塞掉主线程，从而引起ANR，为了避免这样的情况发生，我们引入了异步任务的概念。

异步任务处理的方式有以下多种：

  • RxJava
  • Handler-Looper
  • AsyncTask
  • ……

我个人首推的是这个RxJava，虽然他入门不简单，但是他的代码写起来是链状的，不会嵌套，很干净利索。具体的使用方法，可以在Github上查看文档，有时间我会更新（立Flag）。

Handler这个在之前的介绍中已经深度分析了，所以，这次我的主要任务就是介绍AsyncTask，然后做个简单的对比

##AsyncTask
AsyncTask定义了三种泛型类型 Params，Progress和Result。

Params 启动任务执行的输入参数，比如HTTP请求的URL。
Progress 后台任务执行的百分比。
Result 后台执行任务最终返回的结果，比如String。

一般情况下，我们继承自这个类，然后覆写其中的这几个方法：

  • doInBackground
  • onPreExecute
  • onPostExecute
  • onProgressUpdate
  • onCancelled

使用AsyncTask类，以下是几条必须遵守的准则：

1.Task的实例必须在UI thread中创建；
2.execute方法必须在UI thread中调用；
3.不要手动调用其中的这几个方法，我们需要执行的时候只需要调用execute()方法即可
4.不要多次调用同一个task（有时候当cancel掉时候虽然doInBackground仍在进行，但是不会在调用onProgressUpdate函数了，只能等待线程结束）

###doInBackground
在这个方法当中，我们会进行一些耗时的操作，比如网络访问，数据库访问等等，然后获取数据之后，我们可以调用publishProgress(x)这个方法，将x数据传递给onProgressUpdate函数，来更新我们的实时进度

###onProgressUpdate
这个函数当中，我们可以将我们的数据的更新进度情况设置给我们的ProgressBar当中，让用户知道我们的实时进度

###onPreExecute
这个是在doInBackground之前调用的，进行一些初始化的操作

###onPostExecute
这个函数是在doInbackground之后调用的，我们在这个函数当中，可以将我们获取到的数据更新到我们的UI线程当中，至此，我们的异步操作任务就基本完成了

###onCancelled
这个方法是用来取消掉我们的任务时，我们要进行的操作

观看源码之后，我们可以知道，在这个AsyncTask类的内部，有个线程工厂 

private static final ThreadFactory sThreadFactory = new ThreadFactory() {
    private final AtomicInteger mCount = new AtomicInteger(1);

    public Thread newThread(Runnable r) {
        return new Thread(r, "AsyncTask #" + mCount.getAndIncrement());
    }
};

这也是我们之前讲过的设计模式中的工厂模式

还有一个线程池用来优化系统资源： 

private static class SerialExecutor implements Executor {
    final ArrayDeque<Runnable> mTasks = new ArrayDeque<Runnable>();
    Runnable mActive;

    public synchronized void execute(final Runnable r) {
        mTasks.offer(new Runnable() {
            public void run() {
                try {
                    r.run();
                } finally {
                    scheduleNext();
                }
            }
        });
        if (mActive == null) {
            scheduleNext();
        }
    }

    protected synchronized void scheduleNext() {
        if ((mActive = mTasks.poll()) != null) {
            THREAD_POOL_EXECUTOR.execute(mActive);
        }
    }
}

当我们调用execute方法后，线程池会帮助我们来进行执行我们的任务，到这里可能会有疑问，为什么这个AsyncTask在主线程中创建，可以同时进行耗时和更新UI的矛盾操作呢？因为它是一个新的线程，并且，在内部有个InternalHandler，其实这就是个Handler，并且是主线程，这也就是为什么他可以更新UI，实质上还是使用了Handler-MessageQueue-Looper，将它们包装成了一个异步任务类，使得是用起来代码更加简洁一点。

讲到这里我们的异步任务还是都只是一个线程，每次执行的都是一个任务，那能不能让我们的线程每次执行多个呢？这也和我们的线程池有关，我们可以设置他的线程池为

public static final Executor THREAD_POOL_EXECUTOR

类型的，这样便可以同时执行多个任务了。
